创建型设计模式实战

发表于 2024-08-03 分类于 设计模式
Java 创建型设计模式的实战应用，结合 Spring 源码与业务场景深入理解。

创建型设计模式关注对象的创建机制，提供灵活的对象创建方式，将对象的创建与使用分离。本文深入分析五种创建型模式，结合 Spring 源码与真实业务场景，帮助开发者在实战中正确运用。

单例模式

概念

单例模式确保一个类只有一个实例，并提供全局访问点。适用于需要共享状态或控制资源的场景，如数据库连接池、配置管理器、日志对象等。

UML 类图

核心要素：私有构造方法、静态持有自身实例、公开静态获取方法。

饿汉式

饿汉式在类加载时就完成初始化，线程安全但无法延迟加载。

优点：实现简单，线程安全（由 ClassLoader 保证）。
缺点：无法延迟加载，若实例未使用则浪费资源。

懒汉式

懒汉式在首次使用时才创建实例，支持延迟加载。

DCL（Double-Checked Locking）双检锁

DCL 是懒汉式的高性能版本，避免每次获取实例都加锁。

关键点：volatile 关键字不可或缺。对象创建分为三步：分配内存 -> 初始化对象 -> 引用指向内存地址。若无 volatile，JIT 可能将步骤 2 和 3 重排序，导致其他线程拿到未初始化完成的对象。

枚举实现

Effective Java 推荐的最佳实践，天生防反射、防序列化。

为什么枚举是最好的单例实现：

  1. 线程安全由 JVM 保证（枚举实例在类加载时创建）
  2. 天然防止反射创建新实例（Constructor#newInstance 对枚举有特殊检查）
  3. 天然防止反序列化创建新实例（枚举的 readResolve 由 JVM 实现）

工厂方法模式定义一个创建对象的接口，让子类决定实例化哪个类。将对象创建延迟到子类，实现创建与使用的解耦。

原型模式

概念

原型模式通过复制已有实例来创建新对象，而非通过 new 关键字。适用于创建成本高或构造复杂的对象。

总结

模式 核心意图 典型场景 Spring 应用
单例 保证唯一实例 配置管理、连接池 Bean 默认 Scope
工厂方法 子类决定创建哪个类 框架扩展点 FactoryBean
抽象工厂 创建产品族 多数据库适配 ApplicationContext
建造者 分步构建复杂对象 多参数构造 Security Builder
原型 复制已有实例 创建成本高的对象 Prototype Scope

创建型模式的核心思想是将对象的创建过程封装起来，让客户端代码不依赖具体类的构造方法，从而降低耦合度。在实际开发中，不必刻意使用每种模式，但当场景匹配时，正确运用创建型模式能显著提升代码的可维护性和扩展性。
