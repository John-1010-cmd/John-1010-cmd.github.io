行为型设计模式实战

发表于 2024-09-01 分类于 设计模式
Java 行为型设计模式的实战应用，重点解析策略模式和模板方法在业务系统中的应用。

行为型设计模式关注对象之间的通信和职责分配，描述了对象之间如何协作完成单个对象无法完成的任务。本文深入分析七种行为型模式，结合 Spring 源码和实际业务场景，帮助开发者在日常开发中灵活运用。

策略模式

概念

策略模式定义一系列算法，将每个算法封装到具有公共接口的独立类中，使它们可以互相替换。策略模式让算法的变化独立于使用算法的客户端。

模板方法模式

概念

模板方法在父类中定义算法骨架，将某些步骤延迟到子类实现。子类可以在不改变算法结构的情况下重新定义某些步骤。

扩展方式 继承抽象类 传入 Lambda/接口

观察者模式

概念

观察者模式定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

Spring Event 异步与事务：

  • @EventListener：默认同步执行，与发布者在同一线程
  • @Async + @EventListener：异步执行，需要 @EnableAsync 支持
  • @TransactionalEventListener：在事务特定阶段执行（BEFORE_COMMIT、AFTER_COMMIT 等）

责任链模式

概念

责任链模式将请求的发送者和接收者解耦，使多个对象都有机会处理请求。将这些对象连成一条链，沿着链传递请求，直到有一个对象处理为止。

命令模式

概念

命令模式将请求封装为对象，从而允许用不同的请求对客户端进行参数化、排队请求、记录日志，以及支持撤销操作。

状态模式

概念

状态模式允许对象在内部状态改变时改变其行为，对象看起来像是修改了其类。将状态相关的行为封装到独立的状态类中。

实战场景：订单状态流转

状态模式 vs 策略模式：

特性 状态模式 策略模式
状态转换 状态对象内部管理 客户端控制
客户端感知 不感知状态变化 需要选择策略
行为约束 当前状态限制行为 策略之间独立
典型场景 工作流、状态机 算法选择

迭代器模式

概念

迭代器模式提供一种顺序访问聚合对象中元素的方法，而不暴露其底层表示。将遍历逻辑从聚合对象中分离出来。

Java Iterator

快速失败（fail-fast）机制：迭代器在创建时记录 modCount，每次调用 next() 时检查是否被修改。如果在迭代过程中通过非迭代器方法修改集合（如 list.add()），会抛出 ConcurrentModificationException。

List 遍历方式对比

总结

模式 核心意图 典型场景 Spring 应用
策略 算法封装与替换 支付方式、排序策略 ResourceLoader
模板方法 定义算法骨架 框架流程控制 refresh()、JdbcTemplate
观察者 一对多通知 事件驱动 Spring Event
责任链 请求沿链传递 过滤、拦截 Filter、Interceptor
命令 请求封装为对象 异步执行、撤销 JdbcTemplate
状态 状态驱动的行为变化 订单流转、工作流 -
迭代器 顺序访问聚合对象 集合遍历 Java Iterator

行为型模式是日常开发中使用频率最高的一组模式。其中策略模式和模板方法模式在 Spring 框架中被大量使用，理解这两种模式有助于阅读和扩展 Spring 源码。观察者模式是实现事件驱动架构的基础，在微服务系统中常用于模块间解耦。
